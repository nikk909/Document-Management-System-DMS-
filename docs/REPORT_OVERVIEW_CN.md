# DMS 项目报告与概述

## 项目目标
文档管理系统（DMS）旨在从结构化数据自动生成安全、合规、格式良好的业务文档。

## 业务价值
- **效率提升**: 将文档生成时间从数分钟缩短到数秒。
- **合规保障**: 自动脱敏敏感数据，确保 PII（个人身份信息）保护。
- **品牌一致性**: 水印和集中式模板管理。
- **可追溯性**: 完整记录谁在何时访问或修改了什么。

## 实现亮点

### 1. 混合存储架构
- **对象存储（MinIO）**: 可扩展到数百万文件的高效存储。
- **关系数据库（MySQL）**: 快速搜索和复杂的元数据管理。
- **分离优势**: 元数据与实际文件内容分离存储，提高查询效率和支持大规模文件存储。

### 2. 高级模板引擎
- **Word/HTML 的 Jinja2**: 允许在标准办公文档中使用复杂逻辑、循环和条件格式。
- **VML 集成**: 在 Word 文档中实现高保真背景水印，不影响文本。

### 3. 全面安全保护
- **黑名单执行**: 对文件访问的细粒度控制。
- **多层加密**: PDF 密码 + Word 只读保护。
- **智能数据脱敏**: 上下文感知的敏感字段脱敏。

## 功能实现状态

### ✅ 已完全实现
- 模板管理（上传、分类、检索）
- 文档生成（Word/PDF/HTML，支持 Jinja2 占位符）
- 数据脱敏（自动识别身份证、手机、邮箱、银行卡等）
- 水印功能（文字和图片水印，支持 Word/PDF/HTML）
- 文档加密（PDF 密码、Word 只读保护）
- 访问控制（黑名单权限管理）
- 对象存储（MinIO 集成）
- 元数据管理（MySQL 存储和查询）
- 访问日志（完整的审计跟踪）

### 🚧 部分实现
- **版本控制**: 后端 MinIO 存储和 MySQL 元数据已支持版本历史，但前端 UI 界面尚未完成（缺少版本列表展示和回滚按钮）。

### 📋 未来路线图
- **OCR 集成**: 从扫描的 PDF 中自动提取元数据。
- **工作流审批**: 生成文档的电子签名和审批链。
- **云原生部署**: 完整的 Kubernetes 支持以实现自动扩展。
