# 前端界面文档

基于 HTML/CSS/JavaScript 的前端界面，提供文件管理、模板管理、文档生成等功能。

## 📁 文件结构

```
frontend/
├── index.html    # 主页面
├── app.js        # JavaScript 逻辑
├── styles.css    # 样式文件
└── start_frontend.bat  # Windows 启动脚本
```

## 🚀 快速开始

### 启动前端服务

#### 方法1: Python HTTP 服务器

```bash
cd frontend
python -m http.server 8080
```

#### 方法2: Windows 批处理文件

双击 `start_frontend.bat`

### 访问界面

打开浏览器访问: http://localhost:8080

## 🎯 功能模块

### 1. 登录界面
- 用户登录
- 记住登录状态（Token）

### 2. 文件管理
- 文件列表（支持分页）
- 文件上传（拖拽上传）
- 文件操作：
  - 查看详情
  - 下载
  - 编辑
  - 重命名
  - 删除
  - 归档/取消归档
  - 版本历史
  - 版本回滚
- 文件筛选（日期、类型、标签）
- 一键清空文件

### 3. 模板管理
- 模板列表（支持分页）
- 模板上传
- 模板操作：
  - 查看详情
  - 编辑
  - 版本历史
  - 版本回滚
  - 下载
  - 删除
- 模板筛选（分类、格式）

### 4. 文档生成
- 选择模板
- 选择数据文件（上传新文件或选择已有文件）
- 选择输出格式（Word/PDF/HTML）
- 安全选项：
  - 数据脱敏
  - PDF 加密
  - 水印
  - Word 限制编辑
- 生成文档
- 查看生成的文档列表

### 5. 访问日志
- 日志列表（支持分页）
- 日志筛选（用户、操作、日期）
- 一键清空日志

## 🔧 配置

### API 端点

默认后端 API 地址: `http://localhost:8000`

如需修改，编辑 `app.js` 中的 `API_BASE_URL`：

```javascript
const API_BASE_URL = 'http://localhost:8000';
```

## 🔐 认证

前端使用 JWT Token 进行认证：

1. 登录后，Token 存储在 `localStorage` 中
2. 所有 API 请求自动携带 Token
3. Token 过期后自动跳转到登录页

## 📱 界面特性

- **响应式设计**: 适配不同屏幕尺寸
- **拖拽上传**: 支持文件拖拽上传
- **实时刷新**: 数据自动刷新
- **错误处理**: 友好的错误提示
- **加载状态**: 操作进度提示

## 🎨 样式说明

- **主色调**: 蓝色系（#409EFF）
- **布局**: Flexbox 布局
- **交互**: 悬停效果、过渡动画

## 🐛 故障排除

### 无法连接后端

1. 确认后端服务已启动（http://localhost:8000）
2. 检查浏览器控制台错误信息
3. 检查 CORS 配置

### 登录失败

1. 检查后端服务是否运行
2. 检查用户名和密码是否正确
3. 查看浏览器控制台错误信息

### 文件上传失败

1. 检查文件大小限制
2. 检查文件格式是否支持
3. 查看浏览器控制台错误信息
